# Event-Driven Ansible Roadmap

Event-Driven Ansible is planned for an advanced phase.

## Candidate Use Case

An external monitoring event triggers a remediation workflow.

Example:

1. Alertmanager sends a webhook.
2. A rulebook evaluates the event.
3. EDA launches a controller workflow.
4. The workflow remediates the issue.
5. Notification and ticket status are updated.

## Portfolio Value

This demonstrates event-driven operations, integration architecture, and closed-loop automation.
