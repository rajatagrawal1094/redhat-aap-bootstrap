# Demo Script

The first portfolio demo should tell a clear story.

## Demo Flow

1. Show the GitHub repository as the source of truth.
2. Show config-as-code YAML files.
3. Run lint and validation checks.
4. Apply controller configuration.
5. Show organizations, teams, inventory, project, and job templates in AAP.
6. Register an execution environment.
7. Launch the enterprise Linux patch workflow.
8. Show notification or ticketing behavior.
9. Show validation output and job history.

## Talking Points

- Configuration is repeatable.
- Secrets are separated from Git.
- RBAC is intentional.
- Execution environments make runs reproducible.
- Workflows include validation and operational feedback.
